# CLAUDE.md

Build a production-quality full-stack portfolio website that showcases my experience as an AI and Software Engineer.

The final website will include:

* Responsive frontend
* Backend APIs
* Database
* AI assistant
* Contact form
* GitHub integration
* Calendar booking
* Analytics

Build the project incrementally. Implement only the features requested while designing the codebase so future features integrate cleanly.

## Current Milestone

Focus only on building the frontend portfolio.

Do not implement backend functionality unless I explicitly request it.

### AI

## Theme

Follow the design described in `docs/theme.md`.

The website is a minimal, premium, technical portfolio for an AI & Research Engineer — a high-end startup aesthetic (Apple / Linear / OpenAI / Vercel / Anthropic). Dark, restrained palette, large typography, generous whitespace, and subtle motion. No hiking, trail, or nature metaphors.

## Coding Standards

* Use strict TypeScript.
* Prefer Server Components.
* Use Client Components only when necessary.
* Build reusable components.
* Separate UI, business logic, and data access.
* Keep code modular and maintainable.
* Avoid duplicate code.
* Follow clean architecture principles.

## Styling

* Tailwind CSS only.
* Mobile-first.
* Accessible.
* Smooth, subtle animations.
* Prioritize readability and performance.

## Before Major Changes

For significant architectural changes:

1. Explain the approach.
2. List affected files.
3. Wait for approval.

Small UI improvements and bug fixes can be implemented directly.

## General Principle

Favor simplicity, maintainability, scalability, and clean architecture over clever or overly complex solutions.
